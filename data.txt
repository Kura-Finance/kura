1. Purpose and Scope
The purpose of this Data Retention and Disposal Policy is to ensure that Kura retains user personal and financial data only for as long as is absolutely necessary to provide our services, and that such data is securely and permanently destroyed when no longer needed or upon user request. This policy complies with applicable data privacy regulations and security best practices.

2. Data Classification and Retention Periods
Kura strictly adheres to the principle of data minimization. Our retention schedules are defined as follows:

User Account Data (Name, Email, MFA details): Retained for the active lifespan of the user's account. Upon account deletion request, this data is permanently purged within 30 days.

Financial Data (Bank Balances, Transaction History): Retained only while the user maintains an active connection to their financial institution via Kura.

Plaid Access Tokens & Item IDs: Retained securely in our encrypted database solely to maintain the active data sync. If a user unlinks an institution or deletes their Kura account, the associated Plaid Access Tokens are immediately revoked using Plaid’s /item/remove API endpoint and permanently deleted from our databases.

System Logs & Application Telemetry: Retained for a maximum of 90 days for security monitoring, debugging, and incident response purposes. After 90 days, logs are automatically purged by our cloud infrastructure lifecycle policies.

3. Data Disposal and Destruction Procedures
When data reaches the end of its retention period, or when a user exercises their right to erasure (Right to be Forgotten), Kura enforces the following disposal procedures:

Database Deletion: We perform cryptographic/hard deletions (not soft deletions) for all sensitive financial data and access tokens. Once deleted, the data cannot be recovered.

Third-Party Revocation: We automatically trigger API calls to our data providers (including Plaid) to revoke active access tokens, ensuring no further data can be fetched.

Backups: Data residing in automated encrypted cloud backups will be naturally overwritten or rotated out according to our 30-day backup retention cycle.

4. User Deletion Requests (Right to Erasure)
Users have full control over their data. Kura provides a self-service "Delete Account" and "Unlink Bank" functionality directly within the mobile and web application interfaces. Alternatively, users may submit a formal data deletion request to our security team via email. All deletion requests are processed and finalized within a 30-day Service Level Agreement (SLA).

5. Policy Review and Enforcement
This policy is programmatically enforced through our database schemas and automated backend background jobs. The policy document itself is reviewed and updated at least annually by the engineering and security leadership to ensure ongoing compliance with evolving privacy laws and Plaid's API requirements.
